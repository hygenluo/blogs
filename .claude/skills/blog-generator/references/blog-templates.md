# 博客模板参考

## 1. LeetCode Hot100 题解模板

适用于：用户明确要求写 hot100 博客，或提供的 LeetCode 链接中包含 `top-100-liked`

### Frontmatter
```yaml
---
title: <题目名称>  # 如：两数之和
author: Hygen
description: Leetcode-<题号>  # 如：Leetcode-1
category: hot100
tags: [Solution, <技术标签1>, <技术标签2>]
published: <YYYY-MM-DD>
---
```

### 正文结构
```
# 题目描述
[题目链接](<leetcode.cn 链接>)
<题目原文描述>

## 输入
## 输出
## 约束条件
# 进阶约束 (如有)
## 示例1
## 示例2
...

# 解题思路
<核心思路概述>

## 从暴力枚举出发
<最朴素的做法 + 代码 + 复杂度 + 为什么不够好>

## <逐步优化的思路>
<解释为什么这样优化、怎么做，带代码片段>

## 复杂度
- 时间复杂度：$O(...)$
- 空间复杂度：$O(...)$

## 代码
<最终完整 Python 代码>

# 解题思路（<替代方法名称>）  ← 可选：若有不同算法路线
<替代方法说明 + 代码>

| 对比项 | 方法1 | 方法2 |
| --- | --- | --- |
...

# 代码优化  ← 可选：微优化/代码风格改进
<优化建议>
```

### 关键写作要求
1. **循序渐进**：从暴力解 → 优化解，每一步解释"为什么"
2. **执行过程追踪**：复杂算法用表格展示指针移动过程
3. **复杂度分析**：每个解法都要给出时间/空间复杂度
4. **主动寻找更优解**：写完主解法后，检查是否有更低复杂度或更优雅的替代算法
5. **代码风格**：Python，带类型提示，类 `Solution` 格式
6. **数学公式**：用 `$...$`（行内）或 `$$...$$`（块级）LaTeX


## 2. DSA Solution (普通题解) 模板

适用于：非 hot100 的 LeetCode 题、面试题、算法练习

### Frontmatter
```yaml
---
title: <题目名称>
author: Hygen
description: <题目来源，如 LeetCode-2130 / 华为2025秋招AI岗笔试题>
category: DSA
tags: [Solution, <技术标签>]
published: <YYYY-MM-DD>
---
```

### 正文结构
与 Hot100 模板基本相同，但：
- `description` 中不限于 `Leetcode-NNN`，可写更灵活的描述
- 不需要 `# 题目描述` 的标准化格式（可以更灵活），但建议保留题目链接
- 保持"解题思路 → 代码 → 复杂度"的核心逻辑链

### 文件命名规则
- 紧跟已有编号：如已有 `012."气球"的最大数量`，新文章为 `013.<标题>`
- 文件夹格式：`posts/DSA/Solution/NNN.标题/index.md`


## 3. Agent / LangGraph 技术教程模板

适用于：Agent、LangGraph、RAG 等 AI 框架的技术深度教程

### Frontmatter
```yaml
---
title: <描述性标题>
author: Hygen
description: <一句话描述文章内容>
tags: [LangGraph, Agent]  # 按内容选择
category: Agent  # 或 Guides，按内容深度决定
published: <YYYY-MM-DD>
---
```

### 正文结构
```
# <标题概述>
## <概念1> 是什么
## <概念1> 的作用/优势

# <原理/核心概念> 
<逐步深入讲解>

# <实战/应用>
<完整代码示例>

## 实验结果 / 运行输出
<展示代码运行的实际输出>

# 总结
```

### 关键写作要求
1. 名词首次出现时用中文解释
2. 每个代码示例应完整可运行
3. 建议展示实际运行输出
4. 图构建过程逐步展示，不一次性给完整代码
5. 图片存放于 `assets/` 或 `imgs/` 子文件夹中（参考 SKILL.md 中的附件目录结构），引用格式 `![alt](./assets/filename.png)`


## 4. PyLearning Python 教程模板

适用于：Python 基础语法、编程入门教程

### Frontmatter
```yaml
---
title: Python核心语法<NN>-<主题>
author: Hygen
description: Python语法学习
published: <YYYY-MM-DD>
---
```
注意：PyLearning 文章通常**不加** `tags` 和 `category`

### 正文结构
```
# <编号>. <主题>
## <编号>.<子编号>. <子主题>
### <编号>.<子编号>.<子编号>. 定义
<解释定义>

### <编号>.<子编号>.<子编号>. <相关概念>
<表格展示>  ← 常用表格对比

### 示例
<代码示例 + 输出>

## <编号>.<子编号>. 案例实操
### 案例1: <描述>
<需求 → 代码 → 输出>
```

### 关键写作要求
1. 教科书式结构：定义 → 表格对比 → 代码示例 → 案例实操
2. 使用数字编号层级
3. 每个知识点配代码示例和预期输出
4. 案例实操从实际场景出发


## 5. NLP / 理论知识博客模板

适用于：NLP、Transformer、深度学习等理论知识

### Frontmatter
```yaml
---
title: <系列名>专题<NN>. <主题>
author: Hygen
description: <简要介绍>
tags: [NLP, Transformer]  # 按内容选择
published: <YYYY-MM-DD>
---
```

### 正文结构
```
# <编号>. <问题/概念>是什么？
<通俗解释>

## <概念>的核心<方面>
### 1. ...
### 2. ...

# <编号>. <历史/发展历程>
<Mermaid 时间线>  ← 如适用

# <编号>. <核心原理>
<理论 + 数学公式 + 图示>
```

### 关键写作要求
1. 从"是什么"入手，用通俗语言解释专业概念
2. 善用 Mermaid 图表展示发展历程
3. 数学公式用 LaTeX
4. 引用图片可用 OSS 远程 URL
5. 理论联系实际，说明"为什么重要"

## 8. DSA Knowledge (知识点讲解) 模板

适用于：算法/数据结构的基础概念讲解（非具体题目），如 Kadane 算法、二分查找原理、动态规划入门等

### Frontmatter
```yaml
---
title: <知识点名称>详解  # 如：Kadane 算法详解
author: Hygen
description: <一句话概括知识点>
category: DSA
tags: [<相关技术标签>]
published: <YYYY-MM-DD>
---
```

### 正文结构
```
# 什么是 <知识点>？
<通俗解释，用生活化比喻辅助>

# 核心直觉
> 🧠 **直觉理解**：<一句话抓住本质>

# 从 <角度1> 理解
## 关键定义
## 推导过程
## 代码示例

# 从 <角度2> 理解（可选：不同视角）
...

# 执行过程可视化
<表格展示算法执行过程>

# 变体与扩展
<关联变体、推广方向>

# 总结
<核心收获 + 推荐练习路径>
```

### 关键写作要求
1. **不是题解，是概念讲解**——侧重点在"为什么"而非"怎么做"。可以引用具体题目作为例子，但主线是讲概念
2. **多视角分析**：一个知识点尽量从不同角度解释（如 Kadane 从 DP 和贪心两个视角），帮助读者建立立体的理解
3. **丰富的直观解释**：大量使用 💡 🧠 ⚠️ 提示框，将抽象概念具象化
4. **关联与推广**：在"变体与扩展"中指引后续学习路径
5. **不设编号**：文件夹名直接用英文主题名（如 `kadane`），方便随意扩展
6. **文件夹格式**：`posts/DSA/Knowledge/<主题名>/index.md`
